# Configuration

Orchestra is configured through environment variables loaded from `.env` at the repo
root, plus an optional user-level config file for the CLI.

---

## `.env` file

Created automatically by `make setup` from `.env.example`. All variables are optional
unless marked **required**.

### Database

| Variable | Default | Description |
|----------|---------|-------------|
| `DATABASE_URL` | `postgresql+psycopg://orchestra:orchestra@localhost:5433/orchestra` | SQLAlchemy connection string for the control plane Postgres database |

Postgres runs on host port **5433** (not 5432) to avoid conflicts with other local
Postgres instances.

### LLM provider

| Variable | Default | Description |
|----------|---------|-------------|
| `ANTHROPIC_API_KEY` | *(empty)* | Required for Python loop agents (`backend-agent`, `frontend-agent`, `qa-agent`) and `orchctl ask`/`session` with `llm-backend=python`. Not needed for `claude-code-agent`. |
| `LLM_MODEL` | `claude-sonnet-4-6` | Model used by Python loop agents and `LLMClient` |

### Service URLs

| Variable | Default | Description |
|----------|---------|-------------|
| `ORCHESTRATOR_URL` | `http://localhost:8080` | Base URL for the orchestrator API. Used by `orchctl` and agents. |
| `GATEWAY_URL` | `http://localhost:8081` | Base URL for the gateway API. Used by agents and the dispatcher. |

### Security

| Variable | Default | Description |
|----------|---------|-------------|
| `CAPABILITY_SECRET` | *(generated by setup)* | 32-byte hex secret used to sign capability JWT tokens. Auto-generated by `make setup` if absent or still the placeholder. Must be identical across orchestrator and gateway. |

### Event bus

| Variable | Default | Description |
|----------|---------|-------------|
| `REDIS_URL` | `redis://localhost:6380` | Redis connection URL. Redis runs on host port **6380** (not 6379). |

### Target project

| Variable | Default | Description |
|----------|---------|-------------|
| `SANDBOX_REPO_PATH` | `./sandbox/sample-project` | Absolute path to the Git repo that agents operate on. Written by `make setup` based on your interactive choice. Override before running setup or change in `.env`. |

### Run storage

| Variable | Default | Description |
|----------|---------|-------------|
| `RUN_STORE_DIR` | `/tmp/orchestra/runs` | Directory where context package JSON files are written before each agent run. |

### Observability (optional)

| Variable | Default | Description |
|----------|---------|-------------|
| `OTLP_ENDPOINT` | *(empty)* | OTLP HTTP endpoint for distributed traces (e.g. `http://localhost:4318` for Jaeger). Leave empty to disable tracing. Prometheus metrics are always active at `/metrics`. |

## User-level CLI config

Stored at `~/.config/orchestra/config` (INI format). Managed via `orchctl config`.

```ini
[defaults]
llm-backend = claude
```

### `llm-backend`

Controls which backend `orchctl ask` and `orchctl session` use.

| Value | Behaviour | Prerequisite |
|-------|-----------|-------------|
| `claude` (default) | Launches the `claude` CLI subprocess | `claude` CLI installed + `claude login` done |
| `python` | Uses `LLMClient` (direct Anthropic API call) | `ANTHROPIC_API_KEY` in `.env` |

```bash
orchctl config set llm-backend claude
orchctl config set llm-backend python
orchctl config show
```

The `ORCHESTRA_LLM_BACKEND` environment variable overrides the config file:
```bash
ORCHESTRA_LLM_BACKEND=python orchctl ask backend-agent "what do you know?"
```

## Postgres data persistence

Postgres data is stored at `~/.orchestra/pgdata` (WSL2 bind mount). This avoids the
128 MB Docker Desktop VHD limit.

To wipe all data and start fresh:
```bash
make clean-db
```

This runs: `docker compose down && rm -rf ~/.orchestra/pgdata && docker compose up -d && make migrate`

## Permission files

### `permissions/policy.yaml`

Defines risk-tier assignment rules based on output path globs. The task's overall tier
is the maximum tier across all its output paths.

```yaml
version: 1
rules:
  - pattern: "permissions/**"
    tier: 2
  - pattern: "infra/migrations/**"
    tier: 2
  - pattern: "schemas/**"
    tier: 2
  - pattern: "docs/adr/**"
    tier: 0
  - pattern: "docs/**"
    tier: 0
  - pattern: "tests/**"
    tier: 0
default_tier: 1
```

Tier 2 tasks require `--tier-2-override` to merge. See [ADR-008](../adr/ADR-008-tier-policy.md).

### `permissions/validators.yaml`

The pluggable validator registry. See [Validators](./validators.md) for the full
reference and instructions for adding custom validators.

## `docker-compose.yml`

The compose file starts Postgres and Redis. Services and their configurations:

```yaml
services:
  postgres:
    image: postgres:16
    ports: ["5433:5432"]
    environment:
      POSTGRES_USER: orchestra
      POSTGRES_PASSWORD: orchestra
      POSTGRES_DB: orchestra
    volumes:
      - ~/.orchestra/pgdata:/var/lib/postgresql/data

  redis:
    image: redis:7
    ports: ["6380:6379"]
```

Observability containers (Jaeger, Prometheus, Grafana) are in the `observability`
profile and are not started by default. See [ADR-007](../adr/ADR-007-observability.md).
